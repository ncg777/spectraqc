# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please report it privately.

**Preferred contact:** GitHub Security Advisories

1. Go to the Security Advisories form: https://github.com/spectraqc/spectraqc/security/advisories/new
2. Include a detailed description, steps to reproduce, impact assessment, and any proof of concept.
3. If you have a suggested fix or patch, please include it.

If you are unable to use GitHub Security Advisories, please email: **security@spectraqc.org**.

## Response Expectations

We aim to:

- Acknowledge receipt of your report within **2 business days**.
- Provide a status update within **7 business days** of acknowledgement.
- Work with you to validate the issue and determine a remediation plan.

Please avoid publicly disclosing the issue until we have had a chance to investigate and release a fix.

## Supported Versions

Security fixes are provided for:

- The **latest released minor version** on the most recent major release line.
- The **`main` branch** for unreleased fixes.

Older releases may not receive security updates. If you are running an unsupported version, please upgrade to the latest release.
